# Canon - UI Layout Specification

This document defines the layout, interaction patterns, and component specifications for Canon's user interface.

## Design Philosophy

**"Margin Notes"** â€” A layout inspired by Notion's commenting system and GitHub's code review, optimized for focused code review with minimal friction.

### Core Principles

| Principle | Description |
|-----------|-------------|
| **Code as Hero** | The code viewer is always the primary focus; supporting UI collapses when not needed |
| **Contextual Annotations** | Notes appear alongside the code they reference, not in a separate panel |
| **Zero-Friction Annotation** | Click or drag on line numbers to annotate â€” no modals, no extra steps |
| **Progressive Disclosure** | Summary views are optional, not blocking; complexity appears only when needed |

### State Management

Key state to track:

```typescript
interface LayoutState {
  sidebarCollapsed: boolean
  selectedFile: string | null
  selectedLines: { start: number; end: number } | null
  editingAnnotationId: string | null
  fileAnnotationExpanded: boolean
  summaryPopoverOpen: boolean
}
```

### Gutter Interaction Implementation

The gutter interaction should:

1. Track `mousedown` on line number elements
2. On `mousedown`, record start line
3. On `mousemove` (while mouse down), update end line, show selection highlight
4. On `mouseup`, finalize selection, open annotation input in margin
5. Handle single click (start === end) as single-line annotation
